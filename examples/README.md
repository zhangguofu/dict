# 斐波那契数列计算 - 字典记忆化优化示例

## 1. 问题背景

斐波那契数列定义为：

```
F(0) = 0
F(1) = 1
F(n) = F(n-1) + F(n-2)  (n >= 2)
```

## 2. 方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| 朴素递归 | O(2^n) | O(n) 栈 | 实现简单，但极慢 |
| 记忆化递归 | O(n) | O(n) | 利用 dict 缓存优化 |

## 3. 朴素递归的问题

```
F(5) = F(4) + F(3)
     = F(3) + F(2) + F(2) + F(1)
     = F(2) + F(1) + F(1) + F(0) + F(1) + F(0) + 1
     = ...
```

可以看到 `F(3)` 被计算了 2 次，`F(2)` 被计算了 3 次，随着 n 增大，重复计算呈指数增长。

## 4. 记忆化优化原理

使用字典缓存已计算的结果：

```
计算 F(n) 时:
1. 查缓存 → 命中 → 直接返回
2. 未命中 → 计算 → 存入缓存 → 返回
```

### 代码实现

```c
static dict_handle_t g_fib_cache = NULL;

uint64_t fib_memoized(int n) {
    dict_handle_t cache = fib_get_cache();

    /* 检查缓存 */
    uint64_t result;
    if (dict_get(cache, &n, &result, sizeof(result)) == DICT_OK) {
        return result;  /* 命中 */
    }

    /* 基础情况 */
    if (n <= 1) {
        return (uint64_t)n;
    }

    /* 递归计算 */
    result = fib_memoized(n-1) + fib_memoized(n-2);

    /* 存入缓存 */
    dict_set(cache, &n, &result, sizeof(result));

    return result;
}
```

### 关键设计

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 键类型 | `DICT_KEY_NUMBER` | 使用 int 作为键 |
| 键大小 | `sizeof(int)` | 与键类型匹配 |
| 缓存容量 | 128 | 足够存储 F(0)~F(127) |
| 初始化 | 延迟初始化 | 首次调用时创建 |

## 5. 性能测试结果

测试环境：macOS

| n | F(n) | 朴素递归 | 记忆化递归 | 加速比 |
|---|------|---------|-----------|--------|
| 20 | 6765 | 0.04 ms | ~0.00005 ms | **1,265x** |
| 25 | 75025 | 0.53 ms | ~0.00004 ms | **13,948x** |
| 30 | 832040 | 6.68 ms | ~0.00004 ms | **171,154x** |
| 35 | 9227465 | 62.91 ms | ~0.00003 ms | **185万x** |
| 40 | 102334155 | 658.27 ms | ~0.00003 ms | **2194万x** |

**结论**：n 每增加 5，加速比增长约 **10 倍**。这就是指数级 O(2^n) vs 线性级 O(n) 的差距。
